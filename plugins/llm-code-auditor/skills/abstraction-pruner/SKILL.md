# Abstraction Pruner

Use this skill to remove abstractions that do not earn their complexity.

## Workflow

1. Map each layer in the call graph and write what invariant it adds: mapping, validation, authorization, transaction, caching, retries, lifecycle, observability, or polymorphism.
2. Search for all implementations and callers before editing.
3. Keep boundaries that protect external services, public APIs, tests for hard dependencies, security, concurrency, persistence, or serialization.
4. Collapse layers that only forward unchanged arguments, hold one implementation, or exist for "future flexibility."
5. Rename after collapse using concrete domain nouns.
6. Run tests and type checks that cover the call path.

## Refactor Rules

- Inline helpers called once unless the extracted name is a real domain concept.
- Replace one-implementation interfaces with concrete types unless the boundary is real today.
- Replace factories with direct construction unless object creation is conditional or complex.
- Replace strategy/plugin/event infrastructure with direct calls until a second real participant exists.
- Prefer two clear domain branches over a generic framework for imagined future cases.
